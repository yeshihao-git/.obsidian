# JavaScript 本身只有一个主线程
JavaScript 本身只有一个主线程， **但浏览器 / Node 不是单线程** - 定时器、网络请求、IO、渲染、Worker 都是别的线程 - 它们干完活，把**结果丢到任务队列** - 等主线程空了，再回来执行回调 

**JS 语言层面是单线程，但运行环境是多线程协作；await 是语法糖，不是真的把线程卡住。**

# worker
 1. **Worker 是独立线程** - 可以跑复杂计算，**不会卡住主线程** - 有自己独立的调用栈、堆、事件循环 
 2. **但不能操作 DOM** - 不能碰 `document`、`window` - 不能改页面、不能操作样式 - 只能做纯计算、数据处理 
 3. **通信靠消息传递，不能共享内存** - 主线程 ↔ Worker 用 `postMessage` / `onmessage` - 默认数据是**拷贝传递**，不是直接共享 - 想共享要用 `SharedArrayBuffer`（很少用）
 
一句话总结 **JS 语言本身与控制页面的环境是单线程的； Web Worker 是受限的多线程辅助工具，不能改 DOM、不能共享内存， 所以依然普遍说 JavaScript 是单线程语言。**

# 开发、打包、部署、发布
1. 打包（build / package） **把你写的源代码 → 变成浏览器能直接运行的上线代码** - 你写的：`.vue`、`.tsx`、`scss`、很多零散文件 - 打包后：压缩、合并、编译成纯 `js`/`css`/`html` - 产物一般放在：`dist` 文件夹 工具：vite、webpack、parcel 都是干这个的。
2. 部署（deploy） **把打包好的 dist 文件夹 → 放到服务器上，让别人能访问** 比如： - 传到 nginx - 传到阿里云/腾讯云服务器 - 传到 GitHub Pages、Vercel 等平台 这一步叫**部署**。
3. 发布（publish / release） **让新版本正式对外可用，用户能看到** - 部署完不一定马上对外 - 发布 = 正式上线、对外提供服务 - 库开发者说的发布：把包传到 npm 上让别人下载 

 一句话串起来 1. **写代码** 2. **打包**：源码 → dist 3. **部署**：dist 扔到服务器 4. **发布**：正式对外开放新版本

树摇：打包时自动把没用到的代码 “摇” 掉，只保留实际运行需要的代码。
