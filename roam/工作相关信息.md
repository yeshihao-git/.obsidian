# 天喻软件加密问题

**问题背景**：
```bash
test
├── .vscode
│   └── launch.json          # 调试用配置文件
├── IntevueServer
│   └── IntevueCAE
│       ├── backend          # Java 后端
│       ├── foundation       # C++ 数据处理模块（生成动态库后被 Java 调用）
│       └── vtk-install
└── fronted                  # Vue 前端
    └── src
        ├── Elements
        ├── InterFace
        ├── Operate
        ├── Reader
        ├── appcore
        └── assets
```
在 launch.json 中同时写入了 Vue 前端 和 Java 后端 的调试配置，分别开启两个 vscode （资源管理器中的结构如上所示，都是以 test 为根目录）进行分别调试前后端（前端在 windows 环境中，后端在 wsl 中）。单独调试 Java 后端 没有问题，但启动 Vue 前端 调试的时候，整个后端代码会被加密。
分析原因可能在于 Vue 前端是在 windows 下进行运行调试，而 Java 后端  在 wsl 中；或者由于 Vue 前端代码和 Java 后端代码在一个工作区，Vue 程序运行会导致 Java 代码不安全（例如泄露、窃取等），因此触发了加密。

**解决方式**：
分别启动两个 vscode，前端的资源管理器以 fronted 为根目录，后端的资源管理器以 test 为根目录，并且将 launch.json 文件分别拆分到前后端各自的根目录中进行调试。
