# 跨域问题

**what**：
不允许一个 “源” 去请求另一个 “源”。只要下面三个有一个不同，就是不同源：
1. 协议
2. 域名 / IP
3. 端口

**why**：
浏览器出于安全，防止恶意网站偷偷请求你的后台，偷取数据

**how**：跨域的完整流程
1. 前端发请求 → **浏览器先拦住**
2. 浏览器看：是不是跨域？是  → 检查后端**有没有允许跨域**
3. 后端没开跨域 → 浏览器直接拒绝请求，报：`ERR_CONNECTION_RESET` / `CORS error`
4. 后端开了跨域 → 浏览器放行，请求成功
