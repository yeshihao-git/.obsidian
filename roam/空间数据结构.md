# 1 空间数据结构

计算机图形学涉及的空间数据结构按功能可以划分为两类
1. 空间划分数据结构
2. 几何包围数据结构
两者可以协同工作，先用空间划分数据结构减少需要碰撞检测的物品数量，再用几何包围数据结构减少需要精判物品碰撞的计算量

## 1.1 空间划分数据结构

**what**：
空间为核心，划分整个空间。常见的有：
1. 四叉树
2. 八叉树
3. BSP 树
4. KD 树
5. BVH 树

**why**：
**少找物体**：快速缩小范围，确定需要进行碰撞检测的物品，减少待检测物体总数

> **示例**：场景中有 10000 个模型，检测玩家碰撞
> 只用包围盒：要把玩家 和 10000 个物体，全部逐一做包围盒相交判断。
> 若先进行空间划分：先找玩家所在的空间格子，里面只有 5 个物体；只拿这 5 个和玩家做包围盒判断，剩下 9995 个直接忽略。

| 数据结构      | 划分逻辑                | 使用场景      |
| :-------- | :------------------ | :-------- |
| **四叉树**   | 2D 平面递归四等分          |           |
| **八叉树**   | 3D 立方体递归八等分         |           |
| **BSP 树** | 用分割面递归分割 2D/3D 空间   |           |
| **KD 树**  | 交替沿坐标轴二分分割 2D/3D/高维 |           |
| **BVH 树** | 基于物体包围盒逐层绑定划分 3D 空间 | 光线追踪、碰撞检测 |

## 1.2 几何包围数据结构

**what**：
物体为核心，用简单几何体包裹模型。常见的有：
1. AABB 轴向包围盒
2. OBB 定向包围盒
3. 包围球

**why**：
**少算精细**：快速判断物品是否碰撞（只对包围盒相交的对象再做精细检测），减少计算量

| 类型         | 核心特点             | 优缺点             | 适用场景    |
| :--------- | :--------------- | :-------------- | :------ |
| 包围球        | 球心+半径定义的球体       | 相交检测极快；包裹不紧密    | 粗略碰撞检测  |
| OBB 定向包围盒  | 任意方向的最小包围六面体，可旋转 | 包裹紧密，相交检测慢（开销大） | 精确碰撞检测  |
| AABB 轴向包围盒 | 边与坐标轴对齐的六面体，不可旋转 | 相交检测快，包裹不紧密<br> | 较粗略碰撞检测 |
