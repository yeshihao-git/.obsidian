#操作系统
# 1 虚拟内存
**what：**
虚拟内存是操作系统借助硬盘交换空间，通过 分页/分段 机制，由 MMU 完成逻辑内存地址到物理内存地址的映射；当物理内存不足时，将暂时不用的数据换出到硬盘，在需要时再换回来，实现逻辑上的内存扩容，提供进程地址空间隔离与内存保护。内存映射方式分为：
1. 内存分段
2. 内存分页
3. 段页式

**why：**
1. **安全** ：虚拟内存是进程和物理内存中的一层，进程通过虚拟内存地址访问物理内存，而 虚拟内存地址到物理内存地址的映射是由操作系统完成 的， 操作系统会保证访问不越界 ；现代操作系统通常是多进程的，如果没有虚拟内存，进程可以随意访问物理内存，可能导致程序崩溃 ->  实现方式 ： CPU 中的 MMU、TLB、页表 协作完成
```
没有虚拟内存的情况？
单进程情况：程序1对物理内存的任意地址读写都没问题
多进程情况：程序1在物理内存的地址2000写入新值，程序2又在地址2000写入其他内容
```

2. **效率** ：根据 局部性原理[^1] 我们可以得出结论，程序要运行，无需完整装入内存，只需要装所需的部分，其他部分按需装入或换出 ->  实现方式 ： 虚拟内存 请求调页 、页面置换 机制 
```
没有虚拟内存的情况？
程序太大，装不进内容，无法运行
程序太多，只能运行部分，并发度下降
程序结束前一直驻留内存，浪费内存，并发度下降（程序要运行 只需要部分所需的内容）
```

## 1.1 内存分段
**what：**
按 逻辑 将（虚拟/物理）内存分段（程序：代码段、数据段、堆段、栈段 的逻辑）

**how：** 流程 
CPU将 虚拟内存地址 交给 内存管理单元（MMU），由 MMU 查询 段表 找到 物理内存地址； 具体来说 ：虚拟内存地址分为 段选择因子 、 段内偏移量 ，首先从段选择因子中找到段号，根据段号找到段表中的某一项，段表项中有 段基地址 、 段界限 ，根据段基地址找到物理内存段起始位置，再和虚拟地址的段内偏移量计算得到最终物理内存地址，段界限用于判断内存访问是否越界

![[Pasted image 20260520141619.png|416]]

**存在的问题 ：**
1. 外部碎片（段之间有间隙）
```
3个程序运行，1个程序结束从内存换出，剩余2个程序的段之间出现空隙，第4个程序无法放入这些间隙中，产生了外部碎片
```  

2. 内存交换效率低（要将整个从内存交换到硬盘）

## 1.2 内存分页
**what：**
按 固定大小 划分（虚拟/物理）内存
1. 这个固定大小内存称为 页 ，linux 中页为 4KB
2. 解决内存分段 外部内存碎片（页之间紧密排列）、内存交换效率低（按页交换） 的问题

### 1.2.1 单级页表流程
CPU将 虚拟内存地址 交给 MMU，MMU 查 TLB 若缓冲命中直接返回，否则查询 页表 找到 物理内存地址； 具体来说 ：虚拟内存地址分为页号和页内偏移量，根据页号可以找到对应页表项，页表项中有虚拟页号和物理页号的映射关系，根据物理页号和虚拟内存地址中的页内偏移量找到最终物理内存地址

![[Pasted image 20260520142118.png|406]]

**缺点** ：占用内存空间大 -> 页表需要记录进程完整的内存映射关系 -> 若进程数量很多，则页表在内存中占用就会很大

### 1.2.2 多级页表流程
将虚拟内存中的页号分为一级页号、二级页号（对应一级页表和二级页表），CPU将 虚拟内存地址 交给 MMU，MMU 查 TLB 若缓冲命中直接返回，否则先查询 一级页表，根据 一级页号 找到 二级页表地址，再根据二级页号找到物理页号，最后根据物理页号和虚拟内存地址中的页内偏移量找到最终物理内存地址

![[Pasted image 20260520142159.png|436]]

**优点：**
1. **页表内存占用少**：一级页表覆盖完整内存映射（每个一级页表项表示的范围更大），二级页表按需创建（由于局部性原理，只需要创建部分的二级页表）
```
单级页表不能这样做，因为必须要有完整的内存映射，否则就查不到，多级页表中一级页表覆盖完整的内存映射了
```

2. **查询更快**：
```
从100本书中找到某本书中的一页
单级页表：在100本书中的所有目录中找
多级页表：先在100本书目录中找到那本书，再去书的目录中找那一页
```

### 1.2.3 TLB（快表、页表缓存）
TLB 缓存最常访问的页表项（局部性原理），缓存命中直接返回； 具体来说 ：CPU 将虚拟内存地址 交给 MMU，MMU 查询 TLB，命中则直接使用缓存的页表项，否则常规页表查询

# 2 请求调页
![[Pasted image 20260520143402.png|469]]

**how：** 流程
访问页表时，页表项中的物理页号无效，CPU 发出缺页中断，从磁盘加载页面到物理内存（若没有空闲页，则选择一个物理页换出 -> 页面置换算法），修改页表项的状态位

## 2.1 页面置换算法

| 算法           | 淘汰页面规则                                                                                                                        | 优缺点               |
| ------------ | ----------------------------------------------------------------------------------------------------------------------------- | ----------------- |
| OPT（最佳页面置换）  | 未来最长时间不访问的页面                                                                                                                  | 性能最好；无法实现         |
| FIFO（先进先出）   | 最先进入内存的页面                                                                                                                     | 实现简单；Belady 异常    |
| LRU（最近最久未使用） | 最近一段时间内，最久没访问过的页面                                                                                                             | 性能很好；实现复杂         |
| Clock（时钟置换）  | 将页面放到一个环形链表中，每个页面访问位设置为 1，指针指向最老的页面<br>发生缺页中断时，检查指针指向页面：访问位为 0，则淘汰换入新页面，指针移动到下一个位置<br>访问位是 1 则归 0，指针移动到下一个位置，直到找到访问位为 0 的页面淘汰 | 高效简单（综合 FIFO、LRU） |
| LFU（最少使用）    | 最近一段时间内，访问次数最少的页面                                                                                                             | 实现复杂              |

[^1]: **局部性原理**：
	- **时间局部性**：程序访问某条指令/数据后，不久将再次访问
	- **空间局部性**：程序访问某个存储单元后，附近存储单元很可能被访问
