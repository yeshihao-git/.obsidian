#图形学
**what**：
场景树是一个树形结构，用来**管理3D场景中所有对象的核心数据结构**，定义了谁是谁的父节点、谁跟着谁动、谁包含谁。

> **场景树作用举例**：
> 1. 子节点会继承父节点的位置、旋转、缩放
> 2. 所有对象（模型、标注、灯光、相机）都挂在树上，通过树来控制它们的显示、隐藏、选中、删除

```bash
Root（根节点）
├─ 主3D场景节点
│  ├─ 设备A（父节点） # 移动/旋转设备A时，零件1、零件2会跟着一起动
│  │  ├─ 零件1（子节点，继承设备A的变换）
│  │  └─ 零件2（子节点，继承设备A的变换）
│  └─ 设备B          # 隐藏设备B时，零件3自动一起隐藏
│     └─ 零件3
├─ 2D叠加层节点（THREE2DLayer）
│  └─ 高亮标注
├─ 固定UI层节点（THREELockCameraLayer）
│  └─ PMI信息文本
└─ 相机节点
```
对应的 `Three.js` 代码
```js
// 场景树的根节点
const scene = new THREE.Scene();

// 父节点：设备A
const deviceA = new THREE.Group();
scene.add(deviceA);

// 子节点：零件1、零件2，挂在deviceA下
const part1 = new THREE.Mesh(geometry, material);
const part2 = new THREE.Mesh(geometry, material);
deviceA.add(part1);
deviceA.add(part2);

// 移动父节点，子节点自动跟着动
deviceA.position.x = 10;
```
