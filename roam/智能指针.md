# 1 智能指针

**what**：
封装了原始指针的类对象。分为：
1. std::unique_ptr
2. std::shared_ptr
3. std::weak_ptr

**why**：
是RAII 思想的体现，用类的生命周期自动管理资源，防止内存泄漏


## 1.1 std::unique_ptr

**what**：
独占资源的智能指针，不可拷贝，可移动，通过移动（std::move）转移资源所有权

### 1.1.1 创建unique_ptr

```C++
struct Task {
    Task() { std::cout << "任务开始\n"; }
    ~Task() { std::cout << "任务结束\n"; }
};

int main() {
    // 使用 std::make_unique 创建（推荐方式）
    std::unique_ptr<Task> ptr = std::make_unique<Task>(); 
}
```
- **注意（避坑）：**`make_unique`相比`unique_ptr<Task> p(new Task())`更安全，能防止在复杂表达式中因异常导致的内存泄漏

>**示例**：
>函数调用`process(unique_ptr<Task>(new Task("A")), middle_func())`。 如果 `new Task` 执行完后，在 `unique_ptr` 构造函数执行前，`middle_func()` 抛出了异常，那么 `new` 出来的内存就**泄露**了。而 `make_unique` 保证了创建对象和封装进智能指针是一个原子操作

### 1.1.2 转移资源所有权

```C++
auto p1 = std::make_unique<int>(42);
// std::unique_ptr<int> p2 = p1; // ❌ 编译错误：禁止拷贝

std::unique_ptr<int> p2 = std::move(p1); // ✅ 所有权转移
// 执行后状态：p2 现在指向内存，p1 变成 nullptr
```
- **注意（避坑）：** `unique_ptr` 不允许复制，只能通过 `std::move` 将所有权从一个指针“移动”到另一个指针，在 `move` 之后，通过`if(p)`**检查判空**

### 1.1.3 函数间传递资源

```C++
// 方式 A：转移所有权（函数结束后对象被销毁）
void sink(std::unique_ptr<Task> p) { 
    /* 使用 p */ 
} 

// 方式 B：仅观察/使用（不影响生命周期）
void observe(Task* p) { 
    if (p) p->execute(); 
}

// 调用示例：
auto myTask = std::make_unique<Task>();
observe(myTask.get());       // 传入裸指针
sink(std::move(myTask));     // 彻底交出所有权
```


## 1.2 std::shared_ptr

**what**：
共享资源的智能指针，多个指针共同管理资源，通过引用计数控制资源的生命周期，引用计数为0时销毁资源
1. **底层**：指向资源的指针 + 指向 控制块 的指针
   控制块中有：
	 - **强引用计数**（`use_count`） ：表示管理资源的共享指针数量，管理资源生命周期；
       每增加一个共享指针管理资源： `use_count+1` ，每减少： `use_count-1` ， `use_count=0` 时， `weak_count-1` 销毁资源
     - **弱引用计数**（`weak_count`）：表示观察资源的 std::weak_ptr 数量，管理控制块生命周期；
       每增加一个弱指针观察资源： `weak_count+1` ，每减少： `weak_count-1` ， `weak_count=0` 时，销毁控制块

### 1.2.1 基础创建与引用计数

```C++
#include <iostream>
#include <memory> // 必须包含头文件
#include <string>

struct Book {
    std::string title;
    Book(std::string t) : title(t) {}
};

int main() {
    // 使用 make_shared 分配内存，比 new 更高效且安全
    // 返回值：一个管理 Book 对象的 shared_ptr
    auto ptr1 = std::make_shared<Book>("C++ Primer"); 

    // 拷贝赋值：ptr2 现在也指向同一个对象
    // 内部动作：该对象的引用计数从 1 增加到 2
    std::shared_ptr<Book> ptr2 = ptr1; 

    // ptr1.use_count() 返回当前共享该对象的指针数量
    std::cout << "当前引用计数: " << ptr1.use_count() << std::endl; 

    return 0;
}
```

> **🚀 避坑小贴士：** 始终优先使用 `std::make_shared` 而不是 `std::shared_ptr<T>(new T())`。前者只进行一次内存分配（对象和控制块在一起），性能更好且异常安全。

### 1.2.2 作用域与生命周期管理

```C++
// 当 shared_ptr 离开其作用域时，它会自动减少引用计数。
{
    auto mainPtr = std::make_shared<Book>("Scoped Book");
    {
        auto subPtr = mainPtr; // 计数变为 2
        std::cout << "内部计数: " << mainPtr.use_count() << std::endl;
    } // subPtr 在这里被销毁，计数减 1
    
    std::cout << "外部计数: " << mainPtr.use_count() << std::endl;
} // mainPtr 在这里被销毁，计数减为 0，Book 对象被自动 delete
```

### 1.2.3 循环引用与 weak_ptr

**循环引用**：当两个对象互相持有对方的 `shared_ptr` 时，计数永远不会归零，导致内存泄漏。`std::weak_ptr` 是解决此问题的钥匙。
```C++
struct User;

struct Book {
    // 使用 weak_ptr 观察 User，不增加引用计数，打破循环
    std::weak_ptr<User> reader; 
    ~Book() { std::cout << "Book 释放\n"; }
};

struct User {
    std::shared_ptr<Book> borrowedBook;
    ~User() { std::cout << "User 释放\n"; }
};

int main() {
    auto u = std::make_shared<User>();
    auto b = std::make_shared<Book>();

    u->borrowedBook = b; // User 强引用 Book
    b->reader = u;       // Book 弱引用 User

    // 访问 weak_ptr：必须先 lock() 提升为 shared_ptr
    // 返回值：如果对象还存在，返回有效的 shared_ptr；否则返回 nullptr
    if (auto userPtr = b->reader.lock()) {
        std::cout << "找到借阅者" << std::endl;
    }

    return 0; // 此时 u 和 b 都能正常析构
}
```

> **⚠️ 注意：** `weak_ptr` 没有 `->` 运算符。你必须通过 `.lock()` 检查资源是否依然有效，这保证了在访问时对象不会被意外释放。


## 1.3 std::weak_ptr

**what**：
用于解决 std::shared_ptr 产生的 *循环引用* 问题，是资源的 *观察者* ，不是所有者， *不会增加引用计数*
