# 1 进程

**what：**
运行中的程序， 资源分配 的基本单位，拥有 独立的内存空间 ，考虑 安全 选进程
1. 由 内核调度
2. 进程状态 保存在 进程控制块（PCB）
3. [[进程间的通信方式]]
4. [[进程调度算法]]

![[Pasted image 20260520132713.png|448]]

```
独立内存空间 -> 优： 安全性 较高（进程崩溃不会影响另一个进程）
               缺：进程 切换开销 大（内核参与、需保存整个进程的状态）
```

## 1.1 进程控制块（PCB）

**what：**
进程的唯一标识
1. **进程上下文切换** ：操作系统会保存当前进程A的 CPU状态（存在A的PCB里），然后切换到进程B，执行完B后，保存B的进程CPU状态，恢复A的进程CPU状态
![[Pasted image 20260520133427.png|507]]

2. 操作系统通过 PCB 管理进程，用 链表 将 相同状态的PCB 组织成各种队列
![[Pasted image 20260520133457.png|509]]
```
就绪队列：所有就绪状态的PCB
阻塞队列：所有阻塞状态的PCB
...
```

3. linux中的查看方式： `cat /proc/<PID>/status`

```c
// PCB 中的重要信息
pub struct ProcessControlBlock {
    pid: u16,                           // NOTE 进程标识：区分不同进程
    state: ProcessState,                // NOTE 进程状态：就绪、运行、阻塞等
    // ppid、uid、gid                   // 创建当前进程的父进程ID、所属用户、用户组
    /* 一些寄存器：保存进程的CPU状态 */
    program_counter: u16,               // NOTE 程序计数器：下一条指令的位置
    general_purpose_registers: [u8; 4], // NOTE 通用寄存器：存在正在用于计算的数据
    instruction_register: u8,           // NOTE 指令寄存器：当前正在执行的指令
    // flags: [u1; 3],                  // 标志位
    // stack_pointer: u16,              // 栈指针（可能有）
    // index_registers: [u16; 2],       // 索引寄存器（可能有）
    /* 其他 */
    memory_limits: [u16; 2],            // NOTE 进程的内存范围
    io_devices: Vec<IO_Device>,         // NOTE 打开的IO设备列表
    open_files: Vec<File>,              // NOTE 打开的文件列表
}
```

## 1.2 进程状态

1. **运行**：进程 正在占用CPU
2. **就绪**：进程 等待CPU调用
3. **阻塞**：进程 等待某事件发生（eg：IO完成），此时给它CPU，它也无法运行
4. **创建**：进程 正在被创建
5. **结束**：进程 正在被销毁（eg：运行完成 或 出错）
   （大量的阻塞占用内存，因此有了 挂起状态）
6. **就绪挂起**：进程 在外存（硬盘）等待CPU调度
7. **阻塞挂起**：进程 在外存（硬盘）等待某事件发生


## 1.3 僵尸进程、孤儿进程
### 1.3.1 僵尸进程

**what：**
停止运行，但没被父进程回收的进程，系统中仍保留着 PCB，占用着 PID ，若僵尸进程大量产生，系统无法创建新进程。解决方法：
1. 父进程调用 wait()、waitpid()
2. kill 父进程，让 init进程 收养 僵尸进程，自动调用 wait()回收，释放它的 PCB

### 1.3.2 孤儿进程

**what：**
还在运行，但父进程先于子进程退出的进程；无危害，最后会被 init 进程收养，自动调用 wait() 回收


# 2 线程

**what：**
进程内执行单元， CPU调度 的基本单位， 共享进程内存空间，考虑 性能 选线程
1. 由 内核调度
2. 线程状态保存在 TCB

![[Pasted image 20260520132209.png|353]] 

```
共享进程资源 -> 优： 通信效率 高（共享进程内存）
                    线程 切换开销 小（只需保存线程的状态）
               缺： 安全性 较低（资源竞争、线程崩溃可能导致整个进程终止）
```

## 2.1 线程控制块（TCB）

**what：**
线程的唯一标识
1. **多线程上下文切换** ：操作系统 *保存* 当前线程A的CPU状态到A的TCB里，切换到线程B，执行完后，保存B的线程CPU状态到B的TCB里，从A的TCB *恢复* A的CPU状态

# 3 协程

**what：**
用户态的（程序控制）线程内执行单元（只是代码流程，并不分割CPU资源，协程就是 有状态的函数）， 直接共享所在线程资源，间接共享所属进程资源 
  1. 由 程序调度
  2. 协程状态保存在 程序定义的结构体
  3. 一般用于异步

![[Pasted image 20260520132804.png|387]]

```
程序调度 -> 上下文切换开销极小
```
