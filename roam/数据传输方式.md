# 1 数据传输方式
1. 靠硬盘：一方将数据以临时文件的形式写到硬盘，另一方从硬盘读取临时文件
2. 靠内存：一方将数据变成二进制字节流，另一方直接在内存中读取

# 2 序列化使用场景
1. 传对象 → 必须序列化
2. 传文件 / 流 / 字符串 → 不需要序列化

## 2.1 序列化
**what**：
将内存中的 **对象/结构体** 转成 可传输 的 二进制/字符串。常见的有：
1. JSON
2. XML
3. 二进制字节流

>**注意**：只有在传输 **对象/结构体** 时才需要序列化！
>传文件、二进制流、Base64、字符串 不需要序列化！

**why**：
内存中的对象是临时的，序列化后才能存到 文件、数据库 或 网络传输到其他设备

### 2.1.1 JSON
**what**：
一种数据序列化的具体格式。它的表示如下：
1. 数组 用 `[ ... ]`包裹
2. 对象 用 `{ ... }`包裹
3. 项与项之间用`,`隔开
4. 对象中的键值对用`:`隔开
```json
{
    "FirstName": "John",
    "LastName": "Doe",
    "Age": 43,
    "Address": {
        "Street": "Downing Street 10",
        "City": "London",
        "Country": "Great Britain"
    },
    "Phone numbers": [
        "+44 1234567",
        "+44 2345678"
    ]
}
```
> **避坑**：一个标准的`JSON`文件里，最外层只能有一个`{}`或`[]`

**why**：
便于阅读和编写

> JSON 文件中不能嵌入 二进制数据，需要通过 Base64 将二进制转为 ASCII 字符串，就能放入 JSON 了
