
屏幕坐标、画布坐标

空间和坐标对应关系：
1. **模型空间** → 模型坐标
2. **世界空间** → 世界坐标（3D）
3. **相机空间** → 视图坐标/观察坐标
4. **裁剪空间** → 裁剪坐标（w != 1）
5. **标准化设备空间(NDC)** → 标准化设备坐标（w == 1）
6. **屏幕窗口空间** → 窗口坐标/像素坐标

MVP变换和视口变换
M = Model 模型变换 -> 模型坐标 → 世界坐标
V = View 视图变换（相机）-> 世界坐标 → 相机坐标
P = Projection 投影变换 -> 相机坐标 → 裁剪空间
视口变换

---



## 1. M：Model 模型坐标 → 世界坐标
模型原始局部坐标 → 放到整个场景空间里
平移、旋转、缩放零件，最终得到**世界坐标**。

## 2. V：View 世界坐标 → 相机坐标（视图坐标）
以**相机眼睛**为原点，重新计算所有点的位置。
相当于：站在相机视角看零件，零件现在在哪。

## 3. P：Projection 相机坐标 → 裁剪空间
**最关键一步：3D 压扁成 2D**
透视投影 / 正交投影（工业软件99%用**正交投影**）
把三维空间，投影成正方体规范空间。

## 最后一步：视口变换（Viewport）
标准化坐标 → **屏幕像素坐标**

# 三、完整一条流水线（你渲染面片全过程）
你画一个三角面片，顶点走过的坐标依次是：
$$
\boldsymbol{模型点}
\xrightarrow{M}
\boldsymbol{世界坐标}
\xrightarrow{V}
\boldsymbol{相机坐标}
\xrightarrow{P}
\boldsymbol{裁剪坐标}
\xrightarrow{视口变换}
\boldsymbol{屏幕像素坐标}
$$

一句话总结关系：
> **MVP 就是一套矩阵变换，专门负责把「真实三维世界里的零件点」，一步步换算成「屏幕上的像素点」。**

# 四、结合你工业软件+VTK 通俗大白话版
1. **世界坐标**
零件真实尺寸、毫米单位、三维空间、模型原始位置。
VTK里 `GetPosition()`、面片顶点坐标全是它。

2. **屏幕坐标**
窗口上的像素，鼠标点在哪、窗口xy坐标，二维，没有深度。

3. **MVP 干什么用？**
- 没有MVP：显卡不知道**三维零件**该画在屏幕哪个像素上
- MVP = 坐标转换器
  - M：零件摆在哪
  - V：相机从哪看
  - P：怎么把3D压成2D画面

## 工业软件特别重点：正交投影
CAD、UG、VTK默认都是**正交投影**
- 透视投影：近大远小（游戏、手机3D）
- **正交投影：远近一样大**（工程图纸、测量、剖切、尺寸标注必须用这个）
所以你测量距离、面片拾取、鼠标反求世界坐标，全靠这套MVP逆矩阵。

# 五、开发最实用结论（面试&写代码必背）
1. **世界坐标**：3D真实空间，零件坐标，毫米单位。
2. **屏幕坐标**：2D窗口像素，鼠标坐标，无深度。
3. **MVP本质**：三级矩阵变换，实现

   鼠标给你**屏幕坐标**

   这就是工业软件**点选模型、测量、剖切拾取**的底层原理。

# 一句话区分
1. **浏览器鼠标坐标**：鼠标在**整个网页窗口**里的位置（带滚动、带页面布局）
2. **Canvas 像素坐标**：鼠标在 **Canvas 画布内部**的位置（纯画布坐标系）

---

# 1. 浏览器鼠标坐标
全称：**Browser / Client / Page 坐标**
就是**鼠标相对于浏览器窗口**的位置。

特点：
- 原点：**浏览器窗口左上角 (0,0)**
- 单位：像素
- 范围：整个网页可视区域
- JS 获取：
  - `event.clientX` / `event.clientY` → 相对于**可视窗口**
  - `event.pageX` / `event.pageY` → 相对于**整个网页（含滚动）**

用途：
- 网页菜单、弹窗、拖拽页面元素
- **不是给 Canvas 3D 用的**

---

# 2. Canvas 像素坐标
全称：**Canvas Local Coordinate**
就是**鼠标落在 Canvas 画布内部**的坐标，原点在 Canvas 自己的左上角。

特点：
- 原点：**Canvas 元素左上角 (0,0)**
- 单位：像素
- 范围：只在 Canvas 内部
- 必须通过**浏览器坐标换算**得到

用途：
- Canvas 2D 绘图
- WebGL / Three.js 拾取模型
- **你的工业软件 3D 交互、选面片、测量**

---

# 3. 为什么要区分？（超级重要）
因为：
**浏览器坐标 ≠ Canvas 坐标**

例子：
- Canvas 在页面中间，距离左边 200px
- 鼠标点在 Canvas 左上角
- 浏览器坐标：`(200, 100)`
- Canvas 坐标：`(0, 0)`

所以：
**要在 Canvas 里画画、交互、拾取 3D 模型，必须用 Canvas 坐标！**

---

# 4. 标准换算公式（你项目直接复制）
```js
canvas.addEventListener('mousemove', (e) => {
  // 1. 获取浏览器坐标（相对于窗口）
  const clientX = e.clientX
  const clientY = e.clientY

  // 2. 获取 Canvas 在页面中的位置
  const rect = canvas.getBoundingClientRect()

  // 3. 计算 Canvas 内部像素坐标 ✅
  const canvasX = e.clientX - rect.left
  const canvasY = e.clientY - rect.top
})
```

最终得到的：
- `canvasX, canvasY`
就是标准的 **Canvas 像素坐标**

---

# 5. 极简总结（一秒记住）
- **浏览器鼠标坐标**：鼠标在网页窗口的位置
- **Canvas 像素坐标**：鼠标在 Canvas 内部的位置
- **关系**：Canvas 坐标 = 浏览器坐标 - Canvas 在页面中的偏移

# 6. 对你工业软件的意义（重点）
你做 3D 模型拾取、测量、点击面片时：
1. 浏览器给你：`clientX, clientY`
2. 你转成：`canvasX, canvasY`
3. 再用 MVP 逆矩阵 → 转成**世界坐标**
4. 最终知道点击了**哪个 3D 面片**

这是工业 3D 交互的**标准链路**。

# 四层坐标完整流水线（从上到下完整链路）
我按**鼠标点击的顺序**排，你开发时就是严格走这 4 步：
## 第1层：浏览器窗口坐标（client）
`event.clientX , event.clientY`
- 原点：**浏览器整个窗口左上角**
- 包含页面偏移、div 布局、边距、padding
- 网页原生鼠标事件给你的原始值
- **不能直接用来做3D拾取**

## 第2层：Canvas 像素坐标（真正图形学「屏幕坐标」）
就是你之前 MVP 变换终点的**屏幕坐标**
公式：
$$
\begin{align}
canvasX &= clientX - rect.left \\
canvasY &= clientY - rect.top
\end{align}
$$
- 原点：**Canvas 画布自身左上角**
- 范围：`0 ~ canvas宽`，`0 ~ canvas高`
- **这就是图形学教科书里说的【屏幕坐标】**
- WebGL、Three.js、MVP 变换最终输出的终点坐标

## 第3层：标准化设备坐标 NDC（MVP 内部）
Canvas像素坐标经过**逆归一化**，变成 OpenGL 标准 `-1~1` 立方体坐标
这是 MVP 矩阵工作的中间层。

## 第4层：世界坐标 World
通过 **MVP 逆矩阵**
$$\boldsymbol{M^{-1}V^{-1}P^{-1}}$$
从 Canvas屏幕坐标 → 反推回三维空间
得到模型真实坐标，从而**选中面片、测量、拾取**

---

# 终极对应总结（必背，开发永远不会乱）
1. **浏览器坐标（clientX/clientY）**
   网页DOM坐标，**外部环境坐标**
   带页面偏移、布局距离，**不属于图形学渲染体系**。

2. **Canvas 像素坐标**
   = **图形学原生屏幕坐标**
   MVP变换的最终落点，3D渲染、鼠标拾取的基准坐标。

3. **世界坐标**
   零件模型、面片真实三维坐标
   MVP变换的起点。

4. **MVP 变换**
   负责：
   $$\boldsymbol{世界坐标 \xrightarrow{MVP} Canvas屏幕坐标}$$
   鼠标拾取反过来：
   $$\boldsymbol{Canvas屏幕坐标 \xrightarrow{逆MVP} 世界坐标}$$

---

# 用最通俗一句话串全部概念
> 鼠标在网页上动，浏览器给你**浏览器坐标**；
> 减掉画布偏移，得到**Canvas屏幕坐标**；
> 用MVP矩阵，把三维**世界坐标**渲染到这个屏幕像素上；
> 点击拾取时，再用逆矩阵把屏幕像素反推回**世界坐标**，找到点到了哪个面片。
